##Spring的基本实现
加入maven依赖spring和junit

 <dependencies>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.11</version>
      <scope>test</scope>
    </dependency>

    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-context</artifactId>
      <version>5.2.5.RELEASE</version>
    </dependency>
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-context</artifactId>
      <version>5.2.8.RELEASE</version>
      <scope>test</scope>
    </dependency>
  </dependencies>

2.创建需要创建对象的类模板

    public interface SomeService {
        void doSome();
    }

    public class SomeServiceImpl implements SomeService {

        @Override
        public void doSome() {
            System.out.println("HelloWord");
        }
    }

3.创建spring需要的配置文件，声明类的信息，这些类由spring创建和管理

      <beans xmlns="http://www.springframework.org/schema/beans"
             xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
             xsi:schemaLocation="http://www.springframework.org/schema/beans
             http://www.springframework.org/schema/beans/spring-beans.xsd">
      <!--
          spring的配置文件：
          1.beans:是根标签，spring把Java对象成为bean
          2.spring-beans.xsd：是约束文件
      -->

          <!--
          1.声明bean，就是告诉spring要创建某个类的对象
          id:对象的自定义名称，唯一值。spring通过这个名称来找到对象
          class：类的全限定名称（不能是接口，spring用反射机制创建对象，必须使用类的全限定名称)

          2.spring是把创建好的对象放入map当中，spring框架有一个map存放对象

          -->
          <bean id="someService" class="com.nanye.Service.Impl.SomeServiceImpl"/>

      </beans>

4.测试spring创建的对象

    public class AppTest {


        @Test
        public void shouldAnswerWithTrue() {

            //指定spring容器创建的对象

            //1.指定spring配置文件的名称
            String config = "Spring.xml";

            //2.创建表示spring容器的对象，ApplicationContext
            //ApplicationContext就是表示spring容器，通过容器获取对象
            //ClassPathXmlApplicationContext从类路径加载spring配置文件
            ApplicationContext ac = new ClassPathXmlApplicationContext(config);

            //从容器当中获取某个对象，调用对象方法
            //getBean("配置文件的bean的id“）
            SomeService service=(SomeService)ac.getBean("someService");


            service.doSome();

        }
    }
