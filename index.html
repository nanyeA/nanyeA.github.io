##Mybatis传入参数，传出数据
传入参数
1.parameterType:写在mapper文件当中的一个属性，表示dao接口中方法的参数的数据类型

2.传入参数：从Java代码中把代码中的数据传入mapper文件当中的SQL语句
parameterType:写在mapper文件中的一个属性，表示dao接口中方法的参数的数据类型

3.一个简单类型参数（基本数据类型，String)都叫做简单类型，在mapper类型文件中获取一个参数的值，使用#{}；

4.多个参数，使用@Param命名参数

5.多个参数，使用对象方式传递

传出数据
1.mybatis执行了SQL语句，得到Java对象，类型任意的

2.resultType结果类型，指sql语句完毕后，数据转为的Java对象
处理方式：MySQL执行sql语句，然后mybatis去调用无参构造，创建dx
mybatis把ResultSet指定列值付给同名的属性

3.返回类型为Map,列名是key,列值是value
返回类型是map时只能返回一行数据

4.resultMap:结果映射：指定列名与Java对象的属性对应关系
1.自定义列值赋值给哪个属性
2.当列名与属性名名字一样时，一定使用resultMap

##Mybatis以动态代理实现

1.动态代理定义：为其他对象提供一个代理以控制对某个对象的访问。代理类主要负责为委托了（真实对象）预处理消息、过滤消息、传递消息给委托类，代理类不现实具体服务，而是利用委托类来完成服务，并将执行结果封装处理。
其实就是代理类为被代理类预处理消息、过滤消息并在此之后将消息转发给被代理类，之后还能进行消息的后置处理。代理类和被代理类通常会存在关联关系(即上面提到的持有的被带离对象的引用)，代理类本身不实现服务，而是通过调用被代理类中的方法来提供服务。
引自：https://www.jianshu.com/p/9bcac608c714

2.从上一篇可以发现：dao对象，类型是studentDao，全限定名称：com.nanye.dao.StudentDao，全限定名称和namespace是一样的
方法名称 selectStudents,这个方法就是mapper文件的id值
通过dao方法的返回值确定sqlSession的方法

3.mybatis的动态代理：mybatis根据dao方法调用，获取sql语句信息
mybatis根据你的dao接口，创建出一个dao接口实现类，并创建接口实现类，并创建类的对象
完成sqlSession调用方法，访问数据库

##Mybatis的工具类及其作用和封装自定义工具类
1.Resources:mybatis当中的一个类，负责读取主配制文件

         InputStream in = Resources.getResourceAsStream(config);

2.SqlSessionFactoryBuilder创建SqlSessionFactoryBuilder对象

          SqlSessionFactoryBuilder builder = new SqlSessionFactoryBuilder();

3.SqlSessionFactory类创建SqlSessionFactory对象，SqlSessionFactory重量级对象，创建时间较长，使用资源比较多，
所以一般来说SqlSessionFactory对象有一个就够了，SqlSessionFactory是个接口，SqlSessionFactory接口常用实现类：DefaultSqlSessionFactory
SqlSessionFactory作用：获取SqlSession对象

            SqlSession sqlSession = factory.openSession();

4.openSession()方法说明：无参数的，获取非自动提交事务sqlSession对象
openSession(Boolean):获取自动提交事务的sqlSession

           SqlSessionFactory factory = builder.build(in);

5.SqlSession接口：定义了数据操作的方法（select，insert，delete，update）

          SqlSession sqlSession = factory.openSession();

6.在utils包下创建一个自定义工具类

    public class MybatisUtils {
        private static SqlSessionFactory factory=null;
        static {
            String config = "Mybatis.xml";

            try {
                InputStream in = Resources.getResourceAsStream(config);

                factory=new SqlSessionFactoryBuilder().build(in);
            } catch (IOException e) {
                e.printStackTrace();
            }

        }
        public static SqlSession getSqlSession(){
            SqlSession sqlSession=null;
            if (factory!=null){
               sqlSession= factory.openSession();
            }
            return sqlSession;
        }
    }

7.创建使用mybatis类：访问数据库

 public class MybatisTest01 {
//访问mybatis读取student数据
public static void main(String[] args) {

    //定义mybatis主配制文件的名称(从类路径的根开始target/class）
    String config = "Mybatis.xml";


    try {
        //读取这个config表示文件
        InputStream in = Resources.getResourceAsStream(config);

        //创建SqlSessionFactoryBuilder对象
        SqlSessionFactoryBuilder builder = new SqlSessionFactoryBuilder();

        //创建SqlSessionFactory对象
        SqlSessionFactory factory = builder.build(in);

        //【重要】获取SqlSession对象，从SqlSessionFactory中获取SqlSession
        SqlSession sqlSession = factory.openSession();

        //【重要】指定要执行的sql语句的标识，sql映射文件中的namespace+“."+标签的id值
        String sqlId = "com.nanye.dao.StudentDao" + "." + "selectStudents";

        //执行sql语句，通过sqlId找到语句
        List<Student> studentList = sqlSession.selectList(sqlId);

        //输出结果
        for (Student student : studentList) {
            System.out.println(student);
        }

        sqlSession.close();

    } catch (IOException e) {
        e.printStackTrace();
    }
}
}

##Spring基础理论
1.Spring全家桶，Spring,Springboot,SpringMvc,SpringCloud

2.Spring:为了解决企业级开发的难度，减少对项目模块之间的管理，类和类之间的管理。
帮助开发人员创建对象，管理对象之间的关系

3.Spring核心技术Ioc，Aop（能实现模块之间，类和类之间的解耦合）

4.ClassA中有ClassB的属性和方法，叫做ClassA依赖ClassB

5.ioc概念（控制反转）：是一个描述对象创建，赋值，管理工作都交给代码之外的容器实现的理论
控制：创建对象，赋值，对象之间的管理工作
正转：由开发人员在代码中，使用new构造方法等操作创建对象，开发者自动管理对象
反转：把开发人员管理，创建对象的权限转移给代码之外的容器实现，由容器代替开发人员管理/创建对象

6.容器：是一个服务器软件，一个框架（Spring）

7.ioc的实现：DI技术（依赖注入），只需要在程序中提供要使用的对象名称就可以了，至于如何创建/赋值/查找都是容器内部实现

8.Spring是使用ioc的功能，spring底层创建对象，使用的是反射机制。
